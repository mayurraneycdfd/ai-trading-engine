6. Critical Bugs
These bugs silently inflate backtest results. Fix ALL before trusting any engine output.
Bug 1: Look-Ahead News Leakage
File: features.py:159-167
What: news_sentiment and news_count aggregate ALL news from the SAME day. News published at 14:00 IST feeds into the 09:15 open decision.
Impact: Features news_sentiment, news_count, has_news are all contaminated.
Fix: s = n.groupby("date")["sentiment"].mean().shift(1) — use yesterday's news, not today's.
Bug 2: Portfolio Sharpe Inflation (~5-6× overstated)
File: portfolio.py:85
What: ann_ret = float(daily.mean() * 252) — annualizes over trade-active days only. If strategy trades 50 of 252 days, annual return is inflated 5×.
Impact: Reported Sharpe and annual return are meaningless for selective strategies.
Fix: Reindex daily to full business-day calendar with fill_value=0.0 before annualizing.
Bug 3: Portfolio Leverage Swing
File: portfolio.py:79
What: daily = kept.groupby("date")["net_ret"].mean() — 1 trade on a quiet day = 100% allocation; 10 trades = 10% each. Unrealistic leverage swing.
Fix: daily = kept.groupby("date")["net_ret"].sum() / PORT_MAX_CONCURRENT
Bug 4: Validation Gauntlet Cost Double-Counting
File: validation.py:234-246
What: run_gauntlet nets per-event costs, adds back COST_PCT, then sub-tests subtract COST_PCT again. Some tests double-count costs.
Fix: Compute net returns once at the top; pass pre-netted returns to all sub-tests.
Bug 5: GP Alpha same_sign Always True
File: alpha_search.py:190
What: "same_sign": bool(np.sign(oos_ic) != 0) checks if OOS IC is non-zero, NOT whether IS and OOS IC match.
Fix: "same_sign": bool(np.sign(oos_ic) == np.sign(train_ic))

7. Statistical Weaknesses
7.1 Single Fixed Train/Test Split
File: config.py:44 — TRAIN_END = "2021-12-31"
Problem: All Level 1-4 discovery uses the SAME split. If 2022+ has a different regime, edges that align with post-COVID structure look "confirmed" by accident.
Improvement: Run discovery at 3-4 different TRAIN_END dates; keep only edges confirmed at a majority.
7.2 OOS Confirmation Threshold Too Lenient
File: stats.py:100-113
Problem: Confirmed if OOS same sign AND ≥50% of IS magnitude. For IS mean +0.05%, OOS only needs +0.025% to pass.
Improvement: Add minimum OOS absolute mean (≥0.03%) AND require OOS t-stat ≥ 1.5.
7.3 Quantile Bins Obscure Tail Edges
File: stats.py:33-62
Problem: pd.qcut(factor, 5) forces equal-population bins. Extreme tails (|gap| &gt; 3%) get diluted.
Improvement: Add tail-focused tests at the top/bottom 5%, 3%, 1% independently.
7.4 Level 3 Model Has No CV or Early Stopping
File: edge_discovery.py:149-151
Problem: Hardcoded max_depth=4, learning_rate=0.05, max_iter=300. No validation set, no early stopping.
Improvement: Use early_stopping=True, validation_fraction=0.2; test 2-3 max_depth values.
7.5 Walk-Forward CV Doesn't Actually Purge Label Overlap
File: validation.py:56-86
Problem: Triple-barrier labels look forward up to 120 minutes. Training samples near test fold boundaries can have overlapping labels. The "embargo" is day-based, not label-based.
Improvement: Purge training samples whose label window (TB_MAX_MIN) overlaps the test fold.
7.6 Permutation Test Breaks Within-Day Correlation
File: validation.py:90-126
Problem: On a day when 20 stocks gap due to the same overnight move, outcomes are correlated. The permutation test re-samples within each date, destroying this structure.
Improvement: Shuffle entire date-blocks rather than re-sampling events within dates.
7.7 CSCV/PBO Numerical Edge Cases
File: validation.py:153-195
Problem: When &gt;50% of blocks have &lt;5 events → NaN-dominated matrix → meaningless PBO. Logit transform produces ±inf at rank 0 or 1.
Improvement: Require ≥60% non-NaN blocks; clip logits to [-10, 10].

9. Architectural Gaps
9.1 No Checkpointing / Resume

9.2 No Parallel Processing
Problem: Sequential for-loop over 227 symbols. Feature engineering is CPU-bound and embarrassingly parallel.

9.3 No Memory Management
Problem: All 227 stocks × 11 years of 1-min data loaded into one panel. Can exceed 32GB RAM.

9.4 No Config Validation
Problem: COMBO_MAX_FACTORS=5 with COMBO_TOP_SINGLE=20 → C(20,5) = 15,504 combinations → FDR-diluted.
